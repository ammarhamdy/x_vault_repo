


The de facto archiving utility in Microsoft is the ZIP file. 
ZIP is not as prevalent in Linux but is well supported by the `zip` and `unzip` commands. 
Albeit (ولو), with `tar` and `gzip`/`gunzip` the same commands and options can be used interchangeably to do the creation and extraction, but this is not the case with `zip`. 
The same option has different meanings for the two different commands.


The default mode of `zip` is to add files to an archive and compress it.
```
zip [OPTIONS] [zipfile [file…]]
```


## Recursion `zip -r`:
+ The `zip` command will not recurse into sub-directories by default, which is different behavior than the `tar` command. 
+ That is, merely adding `School` will only add the empty directory and not the files under it. 
+ If you want `tar` like behavior, you must use the `–r` option to indicate recursion is to be used:
```
abc@abc:~/Documents$ zip -r School.zip School
```


## List archived files:
+ The `–l` list option of the `unzip` command lists files in `.zip` archives


## `unzip` command:
+ Extracting the files is just like creating the archive, as the default operation of the `unzip` command is to extract.
+ It gives several options if unzipping files will overwrite existing ones:
```
sysadmin@localhost:~/Documents$ unzip School.zip
```
+ Just like `tar`, you can pass filenames on the command line.
```
sysadmin@localhost:~/Documents/tmp$ unzip School.zip School/Math/numbers.txt
```


# Decompress and Extract


## Decompress

Decompress the `.gz` File
Use the `gunzip` command to decompress the `.gz` part of the file:
```
gunzip file.tar.gz
```
This will produce a `.tar` file (e.g., `file.tar`).


## Extract

Extract the `.tar` File
Use the `tar` command to extract the `.tar` archive:
```
tar -xvf file.tar
```
If you want to simplify things, you can still stick with the `tar -xvzf` command, but splitting it like this can help in debugging or handling specific cases.


**Extract the file** using the `tar` command:
```
tar -xvzf example.tar.gz
```
- `x`  ==Extracts the files.==
- `v`  Displays the process (verbose mode).
- `z`  Handles `gzip` compression.
- `f`  ==Indicates the file name follows.==

**Extract to a specific directory**:
```
tar -xvzf file.tar.gz -C /path/to/destination/
```

**List contents without extracting**:
```
tar -tvzf file.tar.gz
```



# **`Zstandard` (`zstd`)**

Files with the `.tar.zst` extension are compressed tar archives using **`Zstandard` (`zstd`)** compression.

**Using `tar` Directly**

Modern versions of `tar` support `.zst` compression natively. 
You can extract the file in one step:
```
tar --use-compress-program=unzstd -xvf file.tar.zst
```

Or, simply:
```
tar -I zstd -xvf file.tar.zst
```
- `-I zstd`: Specifies the use of Zstandard for decompression.
- `-x`: Extracts the files.
- `-v`: Verbose, shows the extraction process.
- `-f`: Indicates the file name follows.

**Using `zstd` and `tar` Separately**

If your version of `tar` does not support `.zst` directly, you can decompress and extract in two steps:

**Decompress the `.zst` File**:
```
zstd -d file.tar.zst
```

**Extract the `.tar` File**:
```
tar -xvf file.tar
```
